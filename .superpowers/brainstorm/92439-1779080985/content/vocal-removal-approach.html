Vocal Removal — Which approach?

This shapes the whole backend architecture. Pick the tradeoff that fits.

A

FFmpeg Center-Channel Subtraction

Zero new dependencies. Works right now. Uses the stereo mid-side trick: subtracts the center channel (where vocals typically live) from both ears.


Pros

  • Instant — no model download
  • Fast: <5s per track
  • No Python/GPU needed
  • Works offline forever

Cons

  • Mono files unsupported
  • May bleed centered bass/piano
  • Quality ~7/10
B

Demucs AI (Meta Research)

State-of-the-art source separation. Separates vocals, drums, bass, and "other" as clean stems. Industry standard used in professional tools.


Pros

  • Quality: ~9.5/10
  • Works on mono too
  • Clean stems — no bleed
  • Bonus: get drum/bass stems

Cons

  • One-time setup: pip install demucs (~1.5GB model)
  • 30–90s per track (CPU)
  • Requires Python 3.8+
C

Both — Fast fallback + AI upgrade

Backend tries Demucs if installed; falls back to FFmpeg if not. UI shows which engine was used. Best UX — works immediately, gets better when Demucs is set up.


Pros

  • Always works out of the box
  • Upgrades silently when available
  • Progressive enhancement

Cons

  • Slightly more backend code
  • UI needs to show engine quality
