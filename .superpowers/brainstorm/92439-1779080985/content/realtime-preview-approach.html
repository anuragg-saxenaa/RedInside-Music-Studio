Real-Time Preview — How should changes feel?

Currently: trim/speed/volume/fade/reverse are already instant (Web Audio). Reverb/echo/normalize/bass boost/pitch shift require clicking Export to hear. Pick the model.

A

Web Audio API — Instant, always live

All effects run in the browser as you drag sliders. No server call. You hear every change the moment you make it — like a hardware mixer.


Move reverb slider → hear reverb change instantly
Toggle bass boost → hear it in <10ms
Drag pitch knob → pitch changes live
✓ No loading, no spinner, no wait

Pros

  • Truly instant — zero latency
  • No server load for previews
  • Works offline

Cons

  • Browser approximation ≠ exact FFmpeg output
  • Pitch shift needs a JS lib (Tone.js ~200KB)
  • Export still goes through FFmpeg for precision
B

Debounced Server Preview — Exact FFmpeg output

500ms after you stop adjusting, backend processes a short preview clip and auto-plays it. What you hear = exactly what you export.


Move reverb slider → stop → 0.5s delay → hear exact result
Toggle bass boost → 0.5s delay → exact FFmpeg output
⚡ Server generates first 15s preview only (fast)

Pros

  • 100% accurate — no surprises at export
  • Works for all effects including vocal removal

Cons

  • 0.5–2s lag per change
  • Server load on every tweak
  • Needs loading indicator
C

Hybrid — Instant approximation + exact on export

Web Audio API for instant feedback while you're adjusting. FFmpeg for the final export. Best of both: you feel responsive, you export precise.


Drag any slider → hear approximate change instantly
Click Export → FFmpeg produces exact output
Small badge: "Preview ≈ / Export exact"

Pros

  • Feels like pro DAW (Ableton/Logic)
  • No server calls during editing
  • Precise final output

Cons

  • Preview ≠ export exactly (small gap)
  • Most complex to implement
